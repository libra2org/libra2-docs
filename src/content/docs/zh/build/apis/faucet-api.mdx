水龙头允许用户在 Devnet 和 Testnet 上获取测试用的 `APT`.在 Mainnet 上不可用.

## Devnet 和 Testnet 的区别

Devnet 和 Testnet 之间有什么区别?实际上没有.过去,Testnet 的水龙头有一些异常的方式.但现在已经不再如此.
每个水龙头的端点如下:

- Devnet: https://faucet.devnet.libra2labs.com
- Testnet: [todo](/zh/network/faucet)

### 通过终端调用水龙头

如果您想在其他语言中调用水龙头,有两种选择:

1. 从 [OpenAPI 规范](https://github.com/libra2org/libra2-core/blob/main/crates/libra2-faucet/doc/spec.yaml)生成客户端.
2. 自行调用水龙头.

对于后者,您将需要构建类似于以下的查询:

```shellscript filename="Terminal"
curl -X POST
'https://faucet.devnet.libra2labs.com/mint?amount=10000&address=0xd0f523c9e73e6f3d68c16ae883a9febc616e484c4998a72d8899a1009e5a89d6'
```

这意味着铸造 10000 个 [Octas](/zh/network/glossary#Octa) 到地址 `0xd0f523c9e73e6f3d68c16ae883a9febc616e484c4998a72d8899a1009e5a89d6`.

### 调用水龙头:JavaScript / TypeScript

如果您正在使用 JavaScript 或 TypeScript 构建客户端,您应该使用
[@libra2-core/libra2-faucet-client](https://www.npmjs.com/package/@libra2-core/libra2-faucet-client)
包.该客户端是根据水龙头服务发布的 OpenAPI 规范生成的.

Example use:

```typescript filename="index.ts"
import {
  Libra2FaucetClient,
  FundRequest,
} from "@libra2-core/libra2-faucet-client";

async function callFaucet(amount: number, address: string): Promise<string []> {
  const faucetClient = new Libra2FaucetClient({
    BASE: "https://faucet.devnet.libra2labs.com",
  });
  const request: FundRequest = {
    amount,
    address,
  };
  const response = await faucetClient.fund({ requestBody: request });
  return response.txn_hashes;
}
```
