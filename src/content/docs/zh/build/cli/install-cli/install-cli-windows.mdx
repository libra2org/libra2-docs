title: "在 Windows 上安装 Libra CLI"
---

对于 Windows 系统,安装 Libra CLI 工具最简单的方式是通过 PowerShell 脚本.如果该方法不可行,您也可以通过预编译二进制文件手动安装.通常不建议使用预编译二进制文件方式,因为更新过程非常手动化.

# 通过 PowerShell 脚本安装

<Steps>
  1. 在 PowerShell 中运行安装脚本:

     ```powershell filename="Terminal"
     Set-ExecutionPolicy RemoteSigned -Scope CurrentUser; iwr https://docs.libra2.org/scripts/install_cli.ps1 | iex
     ```

  2. 打开新的终端并运行 libra2 help 验证安装是否成功.

     - 您应该能看到 CLI 可运行的命令列表.
     - 这是未来学习每个命令具体用法的有用资源.
</Steps>

<Aside type="note" emoji="ℹ️">
  如需通过脚本将 Libra CLI 更新至最新版本,可以运行 `libra2 update`.
</Aside>

# 通过包管理器安装(可选)

<Aside type="note">
  通过包管理器安装 Libra 后,未来请通过相同包管理器进行更新.
</Aside>

### 如果已安装 [Scoop](https://scoop.sh/),可运行以下命令安装 Libra CLI:

```powershell filename="Terminal"
scoop install https://docs.libra2.org/scoop/libra2.json
```

### 如果已安装 [Chocolatey](https://chocolatey.org/),可运行以下命令安装 Libra CLI:

```powershell filename="Terminal"
choco install libra2
```

### 如果已安装 [winget](https://winget.run/),可运行以下命令安装 Libra CLI:

```powershell filename="Terminal"
winget install libra2
```

# 通过预编译二进制文件安装(备用方案)

<Steps>
  1. 访问 .

  2. 展开 "Assets" 查看预编译二进制文件.

  3. 下载适用于 Windows 的 zip 文件.

     - 文件名格式类似:`libra2-cli-<version>-Windows-x86_64.zip`
     - 下载时可能需要忽略"可疑文件"警告.

  4. 解压下载的文件.

     - 将文件移动至您未来希望调用 `libra2` 的任意文件夹.

  5. 右键点击,然后复制可执行文件路径.

     例如 `C:\Users\<username>\Downloads\libra2-cli-3.1.0-Windows-x86_64\libra2.exe`.

     <Aside type="note" emoji="ℹ️">
       建议将此路径添加至 PATH 环境变量,以便后续简化 Libra CLI 调用.
     </Aside>

  6. 通过开始菜单打开 PowerShell.

  7. 运行 help 命令验证安装.

     使用之前复制的路径调用 Libra CLI.
     例如 `C:\Users\<username>\Downloads\libra2-cli-3.1.0-Windows-x86_64\libra2.exe help`.
</Steps>

<Aside type="note" emoji="ℹ️">
  使用预编译二进制文件安装时,如需更新 Libra CLI,请删除现有安装后重新执行安装步骤.
</Aside>

<Aside type="caution">
  如果以上方法均不奏效,您需要通过源码构建 CLI(请遵循按照以下步骤操作:[安装特定版本的 Libra CLI(高级)](/zh/build/cli/install-cli/install-cli-specific-version)
</Aside>
