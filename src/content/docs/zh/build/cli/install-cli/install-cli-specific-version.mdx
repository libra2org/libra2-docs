---
title: "安装特定版本的 Aptos CLI（高级）"
---

import { Steps } from '@astrojs/starlight/components';

如果需要特定版本的 Aptos CLI,可以直接从 Aptos 源代码构建.此安装方法主要用于与 Devnet 上的特定功能交互,这些功能可能尚未发布到 Testnet/Mainnet.如果您运行的架构与现有发布版本/预编译二进制文件不兼容,也可以按照以下步骤操作.

如果不需要此高级方法,可以在[这里](/zh/build/cli)找到常规安装步骤.

## 在 macOS/Linux 上安装

<Steps>
  1. 按照的步骤操作.

  2. 按照的步骤确保已安装 cargo.

  3. 构建 CLI 工具:cargo build --package aptos --profile cli.

     构建后的二进制文件将位于 `target/cli/aptos`.

  4. (可选)将此可执行文件移动到 PATH 环境变量包含的目录.

  5. 运行 target/cli/aptos help 验证安装是否成功.

     这些帮助说明也作为特定命令的详细指南非常有用.
</Steps>

## 在 Windows 上安装

<Steps>
  1. 按照的步骤从源代码构建 Aptos.

  2. 确保已安装 cargo,可参考的安装步骤.

  3. 构建 CLI 工具:cargo build --package aptos --profile cli.

     构建后的二进制文件将位于 `target\cli\aptos.exe`.

  4. (可选)将此可执行文件移动到 PATH 环境变量包含的目录中.

  5. 运行 target\cli\aptos.exe help 验证安装是否成功.

     这些帮助说明也可作为特定命令的详细使用指南.
</Steps>
