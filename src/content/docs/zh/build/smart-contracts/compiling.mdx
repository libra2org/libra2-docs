## `aptos move compile`

一旦您设置了一个包,您可以通过以下方式编译您的 Move 代码:

```shellscript filename="Terminal"
aptos move compile
```

如果运行成功,您应该会收到如下的终端输出

```shellscript filename="Terminal"
{
  "Result": [
    "<PUBLISHING_ADDRESS>::<MODULE_NAME>"
  ]
}
```

<Aside type="note">
  您可能需要添加命名地址,特别是对于示例.例如,对于 Hello Blockchain Move 示例,您需要添加 `hello_blockchain` 命名地址:

  ```shellscript filename="Terminal"
  aptos move compile --named-addresses hello_blockchain=default
  ```
</Aside>

## 解包构建

编译的 Move 包包含一个类似于下面的文件夹结构.

<FileTree>
  - build/
    - package\_name/
      - bytecode\_modules/
        - dependencies/
        - module\_name.mv
      - source\_maps/
        - dependencies/
        - module\_name.mvsm
      - sources/
        - dependencies/
        - module\_name.move
      - BuildInfo.yaml
  - scripts/
  - sources/
    - module\_name.move
  - tests/
  - Move.toml
</FileTree>

### `bytecode_modules`

字节码模块文件夹包含您的模块的编译 Move 字节码(例如 `module_name.mv`).
要了解有关字节码及其安全功能的更多信息,请参阅 [为什么选择 Move?](/zh/build/smart-contracts/why-move)

### `source_maps`

源映射文件夹包含源映射(例如 `module_name.mvsm`),允许用户将编译的字节码映射回源代码和相关依赖项.
