以下指南提供了在 Libra2 网络上部署 PFN 的步骤.

<Aside type="note">
  **我必须运行 PFN 吗?**<br />

  如果您不想运行 PFN,但仍然想与 Libra2 区块链交互,您可以使用 Libra2 Labs 提供的 REST API (见 [Libra2 API](/zh/build/apis)).不过,请注意,Libra2 Labs 提供的 PFN 有速率限制,这可能会妨碍您的开发.如果您想避免这样的速率限制,您可以部署自己的 PFN 并与 Libra2 区块链直接同步.
</Aside>

选择您喜欢的部署方法:

<Aside type="caution">
  **选择一个网络**<br />

  这些指南默认将 PFN 部署在 Libra2 主网网络上,但它们也可以很容易地在其他网络(例如,devnet 或 testnet 网络)中执行相同的操作.在这样做时,请检出所需的网络分支,并使用相应分支的 `genesis.blob` 和 `waypoint.txt` 节点文件:
  [主网](/zh/network/nodes/configure/node-files-all-networks/node-files-mainnet),
  [devnet](/zh/network/nodes/configure/node-files-all-networks/node-files-devnet) 和
  [testnet](/zh/network/nodes/configure/node-files-all-networks/node-files-testnet) .
</Aside>
