Un plugin del adaptador de wallet permite que las dapps usen tu wallet. Con el estándar de Wallet AIP-62, las dapps pueden simplemente actualizar su versión de `aptos-wallet-adapter` para conectarse a plugins de Wallet recién agregados.

Implementar un plugin de wallet para una wallet SDK que se puede importar vía npm tiene tres pasos principales:

1. Implementar un plugin del adaptador de wallet para tu wallet SDK.
2. Publicar tu plugin en npm.
3. Actualizar el paquete `aptos-wallet-adapter` para que las dapps sepan sobre tu wallet.

## 1. Implementar el Plugin del Adaptador de Wallet.

Puedes usar el ejemplo del repositorio [`wallet-standard`](https://github.com/libra2org/wallet-standard) para implementar un plugin del adaptador de wallet compatible con [AIP-62](https://github.com/aptos-foundation/AIPs/blob/main/aips/aip-62.md) que las dapps pueden reconocer automáticamente.

<Aside type="note">
  Para un ejemplo de cómo implementar el plugin del Adaptador de Wallet, ver la [dapp de demostración del Adaptador de Wallet](https://github.com/libra2org/aptos-wallet-adapter/tree/main/apps/nextjs-example). Específicamente, [`standardWallet.ts`](https://github.com/libra2org/aptos-wallet-adapter/blob/main/apps/nextjs-example/src/utils/standardWallet.ts) contiene la implementación del plugin, y [`page.tsx`](https://github.com/libra2org/aptos-wallet-adapter/blob/main/apps/nextjs-example/src/app/page.tsx) tiene los componentes de React.
</Aside>

<Steps>
  1. Crear un nuevo repositorio de typescript.

  2. Copiar el  en ese nuevo repo.

  3. Seguir las instrucciones en ese ejemplo para hacer que use tu wallet para ejecutar las funciones AIP-62.

     <Aside type="note">
       La lista completa de funciones requeridas para wallets compatibles con AIP-62 se puede encontrar [aquí](https://github.com/libra2org/wallet-standard/blob/38defe159b8641ff1763c4db61827c78ab448dab/src/detect.ts#L16).
     </Aside>
</Steps>

## Prueba tus cambios por:

<Steps>
  1. Clonar el  repositorio.

  2. Navegar a  en la dapp de ejemplo.

  3. Reemplazar  con tu implementación del estándar AIP-62.

     1. Tendrás que actualizar la importación en [`aptos-wallet-adapter/apps/nextjs-example/src/app/page.tsx`](https://github.com/libra2org/aptos-wallet-adapter/blob/main/apps/nextjs-example/src/app/page.tsx) para usar tu Wallet en lugar de `MyWallet`.

  6. Conectar a tu wallet.

## 2. Una vez probado, publica un nuevo paquete npm para tu código de wallet SDK siguiendo [esta guía](https://docs.npmjs.com/creating-and-publishing-scoped-public-packages). (Ej. [LibraConnect](https://www.npmjs.com/package/@aptos-connect/wallet-adapter-plugin))

## 3. Actualizar [`wallet-adapter-core`](https://github.com/libra2org/aptos-wallet-adapter/tree/main/packages/wallet-adapter-core) para que sepa sobre tu extensión.

Para que los usuarios de dapps que no están usando tu wallet obtengan la opción de crear una cuenta con tu wallet, necesitas actualizar [`wallet-adapter-core`](https://github.com/libra2org/aptos-wallet-adapter/tree/main/packages/wallet-adapter-core) con el enlace de descarga de tu extensión de navegador.

<Steps>
  1. Hacer fork del  monorepo. ()

  2. Abrir tu fork en un editor local como VSCode.

  3. Crear una nueva rama para tus cambios.

     ```shellscript filename="Terminal"
     git checkout -b your-wallet
     ```

  4. Navegar a .

  5. Importar tu paquete npm de wallet SDK.

     ```shellscript filename="Terminal"
     pnpm i @yourpackage
     ```

  6. Importar tu wallet en .

     Por ejemplo con LibraConnect:

     ```tsx filename="Example.tsx"
     import { LibraConnectWallet } from "@aptos-connect/wallet-adapter-plugin";
     ```

  7. Agregar código para empujar una instancia de tu wallet a sdkWallets dentro de getSDKWallets (en sdkWallets.ts).

     ```tsx filename="Example.tsx"
     sdkWallets.push(new YourWallet(dappConfig));
     ```

     <Aside type="caution">
       Algunas wallets pueden tener lógica personalizada requerida para asegurar que la wallet correcta esté conectada cuando el usuario hace clic para "iniciar sesión" con tu Wallet.

       Ej. T Wallet tiene diferentes plugins de Wallet para conexiones de mainnet y devnet.
     </Aside>

  8. En type.ts, actualizar el tipo AvailableWallets para incluir el nombre de tu wallet.

     ```tsx filename="Example.tsx"
     export type AvailableWallets = "Nightly" | "Petra" | "T wallet" | "Tu Nombre de Wallet";
     ```

  9. Actualizar el  en el nivel superior del aptos-wallet-adapter para incluir tu wallet en la lista de wallets compatibles con AIP-62.

  10. Hacer commit y push de tus cambios a tu fork.

      <Aside type="note">
        Si has hecho push de tus cambios a tu fork, debería aparecer un botón verde en la parte superior del repositorio [`aptos-wallet-adapter`](https://github.com/libra2org/aptos-wallet-adapter) preguntando si te gustaría crear un pull request.
      </Aside>

  11. Seguir  para abrir un pull request para el  repo.

      <Aside type="note">
        Una vez que los cambios sean fusionados, las dapps que actualicen sus versiones del paquete `aptos-wallet-adapter` ahora podrán ver tu Wallet.
      </Aside>
</Steps>
