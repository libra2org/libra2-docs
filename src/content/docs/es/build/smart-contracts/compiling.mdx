## `aptos move compile`

Una vez que tengas un paquete configurado, puedes compilar tu código Move haciendo:

```shellscript filename="Terminal"
aptos move compile
```

Si se ejecuta exitosamente, deberías recibir una salida de Terminal como esta:

```shellscript filename="Terminal"
{
  "Result": [
    "<DIRECCIÓN_DE_PUBLICACIÓN>::<NOMBRE_DEL_MÓDULO>"
  ]
}
```

<Aside type="note">
  Puede que necesites agregar direcciones nombradas, especialmente para ejemplos. Por ejemplo, con el ejemplo Hello Blockchain Move, necesitarás agregar la dirección nombrada `hello_blockchain`:

  ```shellscript filename="Terminal"
  aptos move compile --named-addresses hello_blockchain=default
  ```
</Aside>

## Desempaquetando la Compilación

Los paquetes Move compilados contienen una estructura de carpetas que se asemeja a la siguiente.

<FileTree>
  - build/
    - package\_name/
      - bytecode\_modules/
        - dependencies/
        - module\_name.mv
      - source\_maps/
        - dependencies/
        - module\_name.mvsm
      - sources/
        - dependencies/
        - module\_name.move
      - BuildInfo.yaml
  - scripts/
  - sources/
    - module\_name.move
  - tests/
  - Move.toml
</FileTree>

### `bytecode_modules`

La carpeta de módulos de bytecode contiene el bytecode Move compilado para tu(s) módulo(s) (como `module_name.mv`).
Para aprender más sobre el bytecode y sus características de seguridad, consulta [¿por qué move?](/build/smart-contracts/why-move)

### `source_maps`

La carpeta de mapas de fuente contiene mapas de fuente (como `module_name.mvsm`) que permiten
a los usuarios mapear el bytecode compilado de vuelta al código fuente y dependencias relevantes.
