<Aside type="note">
  Recomendamos instalar la CLI de Libra antes de comenzar.
  Si aún no has instalado la CLI de Libra, consulta la [sección CLI](/build/cli)
</Aside>

<Steps>
  1. libra2 move init

     En un nuevo directorio de proyecto, inicializa un paquete Move ejecutando:

     ```shellscript filename="Terminal"
     libra2 move init --name <NOMBRE_DEL_PROYECTO>
     ```

     Ahora deberías tener un proyecto Move que se ve así:

     <FileTree>
       - scripts/
       - sources/
       - tests/
       - Move.toml
     </FileTree>

     <Aside type="note">
       También puedes crear un paquete Move desde una [plantilla](/build/cli/start-from-template).
     </Aside>

  2. Actualizar Move.toml

     En `Move.toml`, completa la siguiente información clave:

     1. `name`: nombre de tu paquete
     2. `version`: versión del paquete (por defecto es `"0.0.0"`)
     3. `addresses`: Describe a qué dirección se desplegará el módulo. Estas son direcciones nombradas que se pueden usar como alias. En el ejemplo de abajo, usaremos `hello_blockchain` como la dirección nombrada.
     4. `dependencies`: Probablemente querrás usar `LibraFramework` y otras [Dependencias de Terceros](/build/smart-contracts/third-party-dependencies)

     Abajo hay un ejemplo

     ```toml filename="Move.toml"
     [package]
     name = "Examples"
     version = "0.0.0"

     [addresses]
     hello_blockchain = "_"

     [dependencies.LibraFramework]
     git = "https://github.com/libra2org/libra2-framework.git"
     rev = "mainnet"
     subdir = "libra2-framework"
     ```

  3. Agregar al directorio sources

     Agrega tu código en el directorio `sources`. Aquí tenemos un ejemplo `hello_blockchain.move`.

</Steps>
