Este tutorial introduce un caso de uso práctico que combina la cuenta multisig del framework de Libra con el estándar de activo fungible para mejorar el margen de seguridad en la gestión de activos fungibles. Asegúrate de haber entendido la publicación de módulos y la cuenta multisig del framework de Libra antes de continuar con el tutorial. Si no es así, se recomienda encarecidamente probar primero los siguientes tutoriales:

- [Tu Primer Módulo Move](/build/guides/first-move-module)

## Paso 1: Elige un SDK

Este tutorial fue creado para el [SDK de TypeScript](/build/sdks/ts-sdk).

¡Otros desarrolladores están invitados a agregar soporte para el [SDK de Python](/build/sdks/python-sdk), [SDK de Rust](/build/sdks/rust-sdk), [SDK de Go](/build/sdks/go-sdk) y [SDK de Unity](/build/sdks/unity-sdk)!

## Paso 2: Publica el módulo

Para crear un activo fungible controlado por una cuenta multisig del framework de Libra con todas las operaciones administrativas (acuñar, transferir, quemar, congelar/descongelar), es un requisito previo un contrato inteligente bien diseñado basado en el estándar de activo fungible. El equipo de Libra proporciona un código de ejemplo en el repositorio `libra2-core`.

Clona el repositorio `libra2-core`:

```shellscript filename="Terminal"
git clone git@github.com:libra2-core/libra2-core.git ~/libra2-core
```

Navega al directorio `managed_fungible_asset` y luego publica este paquete en tu cuenta `default` usando la CLI:

```shellscript filename="Terminal"
cd ~/libra2-core/libra2-move/move-examples/fungible_asset/managed_fungible_asset
libra2 move publish --named-addresses example_addr=default
```

Navega al directorio `multisig_managed_coin` y luego publica este paquete en tu cuenta `default` usando la CLI también:

```shellscript filename="Terminal"
cd ~/libra2-core/libra2-move/move-examples/fungible_asset/multisig_managed_coin
libra2 move publish --named-addresses example_addr=default
```

Para este tutorial, `multisig_managed_coin` necesita llamar funciones definidas en `managed_fungible_asset` en la misma dirección. Así que ambos módulos deben ser publicados.

<Aside type="note">
  No olvides financiar la cuenta con faucet antes de publicar los módulos.
</Aside>

## Paso 3: Inicia el ejemplo

```shellscript filename="Terminal"
cd ~/libra2-core/ecosystem/typescript/sdk/examples/typescript
```

Ejecuta el ejemplo `multisig_managed_coin`:

```shellscript filename="Terminal"
MODULE_ADDR=${DEFAULT_ACCOUNT_ADDRESS} pnpm run multisig_managed_coin
```

<Aside type="note">
  Este ejemplo usa la devnet de Libra, que históricamente se ha reiniciado cada jueves.
  ¡Asegúrate de que devnet esté activa cuando intentes ejecutar el ejemplo! Si estás ejecutando localnet con faucet, puedes ejecutar el siguiente comando en su lugar:

  ```shellscript filename="Terminal"
  export APTOS_NODE_URL=http://0.0.0.0:8080
  export APTOS_FAUCET_URL=http://0.0.0.0:8081
  export MODULE_ADDR=${DEFAULT_ACCOUNT_ADDRESS}
  pnpm run multisig_managed_coin
  ```
</Aside>

El script de ejemplo debería ejecutarse correctamente sin errores. Luego podrás ver lo que hizo buscando las direcciones `owner1` y `owner2` impresas en la consola en el explorador de Libra.

## Conclusión

Este tutorial muestra un flujo end-to-end de usar una cuenta multisig del framework de Libra para administrar activos fungibles. De manera similar, puedes crear tu propio módulo y aprovechar nuestro poderoso SDK para crear el esquema de administración que se ajuste a tus necesidades.
