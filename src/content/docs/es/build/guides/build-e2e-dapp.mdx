Una forma común de aprender un nuevo framework o lenguaje de programación es construir una lista de tareas simple. En este tutorial, aprenderemos cómo construir una dapp de lista de tareas end-to-end, comenzando desde el lado del smart contract hasta el lado del frontend y finalmente el uso de una wallet para interactuar con ambos.

Ve el código completado en el [código fuente](https://github.com/libra2org/developer-docs/tree/main/apps/nextra/pages/en/build/guides/build-e2e-dapp).

## Prerrequisitos

Debes tener:

- [node y npm](https://nodejs.org/en/)

Aunque explicaremos algunas decisiones de React, no vamos a profundizar en cómo funciona React; así que asumimos que tienes alguna experiencia previa con React.

## Configuración

En esta sección, crearemos un directorio `my-first-dapp` para contener nuestros archivos de proyecto, tanto código del lado del cliente (basado en React) como el código Move (nuestro smart contract).

Para eso, usaremos [create-aptos-dapp](/build/create-aptos-dapp) para crear el proyecto.

1. Abre una terminal y navega al directorio deseado para el proyecto (por ejemplo, el directorio `Desktop`).
2. Ejecuta `npx create-aptos-dapp@latest` para crear el proyecto.

```shellscript filename="Terminal"
npx create-aptos-dapp@latest
```

3. Sigue las instrucciones para crear el proyecto.

- Elige un nombre para el proyecto, por ejemplo `my-first-dapp`.
- Elige la opción `Full-stack project`.
- Elige la opción `Boilerplate Template`.
- Por simplicidad, elige no usar Surf.
- Elige la opción de framework `Vite app`.
- Elige la opción de red `Devnet`.

La herramienta creará el proyecto en un directorio con el mismo nombre que el proyecto e instalará las dependencias requeridas.

Sigue las instrucciones de `Próximos Pasos`.

Ahora vamos a [crear un smart contract](/build/guides/build-e2e-dapp/1-create-smart-contract).
