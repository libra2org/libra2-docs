Si estás ejecutando tu propia instancia de la [API del Indexador](/build/indexer), o un procesador personalizado del [SDK del Indexador](/build/indexer/indexer-sdk), debes tener acceso a una instancia del Servicio de Stream de Transacciones. Esta página contiene información sobre cómo usar el Servicio de Stream de Transacciones Alojado por Libra2 Labs.

## Endpoints

Todos los endpoints están en GCP us-central1 a menos que se especifique lo contrario.

- **Mainnet:** grpc.mainnet.libra2labs.com:443
- **Testnet:** grpc.testnet.libra2labs.com:443
- **Devnet:** grpc.devnet.libra2labs.com:443

Puedes aprender sobre los límites de tasa para este servicio leyendo la [documentación de Libra2 Build](https://build.libra2labs.com/docs/start/billing).

## Autorización vía API Key

Para usar el Servicio de Stream de Transacciones Alojado por Labs debes tener una API key. Para obtener una API key, haz lo siguiente:

1. Ve a https://build.libra2labs.com.
2. Inicia sesión y selecciona "API Resource".
3. Crea una nueva key. Verás el secreto de la API key en la primera tabla.

Puedes proporcionar la API key configurando el header HTTP `Authorization` ([MDN](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Authorization)). Por ejemplo, con curl:

```shellscript filename="Terminal"
curl -H 'Authorization: Bearer libra2labs_yj4donpaKy_Q6RBP4cdBmjA8T51hto1GcVX5ZS9S65dx'
```

Aprende más sobre las API keys en el [sitio de documentación de Libra2 Build](https://build.libra2labs.com/docs/start/api-keys).
