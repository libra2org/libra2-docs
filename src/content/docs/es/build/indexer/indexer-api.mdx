Aptos Labs aloja una versión pública de la API GraphQL del Indexador que cualquiera puede usar para obtener datos históricos y agregados básicos sobre transacciones, activos fungibles y tokens de la cadena.

Puedes explorarla manualmente viendo el Hasura Explorer a continuación para la red que te interese.

También puedes acceder a la API a través de los endpoints GraphQL a continuación. Para más información sobre el formato de datos en cada campo/tabla, consulta la [página de referencia de tablas](/build/indexer/indexer-api/indexer-reference).

## Acceso SDK (Método Principal)

La forma principal de usar el Indexador es acceder a él a través del [SDK de TypeScript](/build/sdks/ts-sdk/fetch-data-via-sdk).

El SDK de TypeScript manejará automáticamente los límites de tasa y puede permitir sin problemas tanto el acceso a la [API REST del Fullnode](/build/apis/fullnode-rest-api) como el acceso al Indexador dependiendo de qué datos se necesiten.

## Endpoints de la API GraphQL (Acceso Directo)

Si necesitas hacer consultas GraphQL directamente a la API del Indexador alojada por Aptos-Labs, entonces usa los siguientes endpoints:

- **Mainnet:** `https://api.mainnet.aptoslabs.com/v1/graphql`
- **Testnet:** `https://api.testnet.aptoslabs.com/v1/graphql`
- **Devnet:** `https://api.devnet.aptoslabs.com/v1/graphql`

### Límites de tasa

Aprende más sobre los límites de tasa que se aplican a la API del indexador alojada por Aptos Labs leyendo la [documentación de Aptos Build](https://build.aptoslabs.com/docs/start/billing).

Si necesitas un límite de tasa más alto, considera las siguientes soluciones:

1. Obtén una API Key de [Aptos Build](https://build.aptoslabs.com/). Aprende más sobre las API keys en el [sitio de documentación de Aptos Build](https://build.aptoslabs.com/docs/start/api-keys).
2. Ejecuta la API del Indexador de Aptos tú mismo. Consulta la guía para auto-alojamiento [aquí](/build/indexer/txn-stream/self-hosted).
